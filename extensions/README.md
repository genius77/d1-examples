# D1 Extensions — 扩展机制

D1 提供三种扩展机制，允许用户在动态库基础上编写自定义业务逻辑：

| 类型 | 目录 | 说明 | 生命周期 |
|------|------|------|----------|
| **Script** | `script/` | JavaScript 脚本，D1 内置 JS 引擎执行 | 即调即执行 |
| **Exec** | `exec/` | 外部可执行文件（Shell / Binary） | 即调即走 |
| **Service** | `service/` | 常驻微服务进程 | D1 启动时拉起，退出时停止 |

## 扩展机制架构

```
┌────────────────────────────────────────────────┐
│                   D1 动态库                     │
│                                                │
│  ┌──────────┐  ┌──────────┐  ┌──────────────┐ │
│  │ Script   │  │  Exec    │  │   Service    │ │
│  │ Engine   │  │  Runner  │  │   Manager    │ │
│  │ (Goja)   │  │          │  │              │ │
│  └────┬─────┘  └────┬─────┘  └──────┬───────┘ │
└───────┼──────────────┼───────────────┼─────────┘
        │              │               │
   ┌────▼────┐   ┌─────▼─────┐  ┌──────▼──────┐
   │ .js 脚本 │   │ .sh/exe   │  │ 微服务进程   │
   │ 内置执行 │   │ 独立进程   │  │ 独立进程    │
   └─────────┘   └───────────┘  └─────────────┘
```

## 目录结构

```
extensions/
├── README.md                 # 本文件
├── script/                  # JavaScript 脚本示例
│   ├── example.js            # 展示所有 d1.* API 用法
│   ├── custom_handler.js      # 自定义消息处理
│   └── data_transform.js     # 数据转换
├── exec/                    # 外部可执行文件示例
│   ├── data_converter.sh     # 数据格式转换
│   ├── status_check.sh       # 状态检查
│   └── backup.sh             # 备份脚本
└── service/                 # 常驻微服务示例
    ├── cache_service/         # 缓存服务（Go 示例）
    └── rule_engine/           # 规则引擎（Python 示例）
```

## 配置关联

扩展需要在 D1 配置文件中注册后才能使用。对应关系：

| 扩展类型 | 配置文件 |
|----------|----------|
| Script | `script.yaml` — 脚本路径 + 超时 + 内存限制 |
| Exec | `exec.yaml` — 可执行文件路径 + 并发数 |
| Service | `service.yaml` — 启动命令 + gRPC 配置 |

> 完整配置示例见仓库根目录 `config/` 目录。
